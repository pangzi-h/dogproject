# 多智能体联动使用模板

本文件为项目启动与协作执行模板，配合多智能体联动协议使用。

## 使用说明
1. 每次新项目从 1.1 开始顺序发送。
2. 文本中的方括号字段需替换为真实项目信息。
3. 若阶段失败，按回归修复模板回退状态后重走提测流程。

## 1. 启动与执行模板

### 1.1 首次启动模板（发给产品智能体）
```text
请按产品智能体模式启动本项目。
项目名称：[项目名称]
版本：[版本号]
目标上线时间：[日期]
输入资料：[PRD链接或本地路径]
补充说明：[业务目标/范围边界]

要求：
1. 生成标准版PRD（P0/P1/P2优先级）。
2. 完成中国+海外竞品对比。
3. 我确认后，再通知前端开发智能体与后端开发智能体启动。
```

### 1.2 前后端启动模板（由产品智能体发送）
```text
【需求已确认，启动开发】
项目名称：[项目名称]
版本：[版本号]
模块：[模块名]
当前状态：需求中 -> 方案中
PRD文档：[路径或链接]
优先级：[P0/P1/P2]
截止时间：[日期]
风险说明：[可为空]
请开始系分与技术方案设计。
```

### 1.3 前端执行模板（发给前端开发智能体）
```text
请按前端开发智能体模式执行：
项目名称：[项目名称]
模块：[模块名]
PRD文档：[路径或链接]
原型图：[路径或链接]
技术栈：[React/Vue/uni-app]
截止时间：[日期]

要求：
1. 先输出前端系分文档。
2. 系分确认后进入开发。
3. 联调通过后按协议模板发起提测。
```

### 1.4 后端执行模板（发给后端开发智能体）
```text
请按后端开发智能体模式执行：
项目名称：[项目名称]
模块：[模块名]
PRD文档：[路径或链接]
项目类型：[迭代/全新]
技术栈：[Java Spring Boot 默认]
截止时间：[日期]

要求：
1. 先输出后端系分文档与接口文档。
2. 系分确认后进入开发与单测。
3. 联调通过后按协议模板发起提测。
```

## 2. 联调与提测模板

### 2.1 联调申请模板（前后端互发）
```text
【接口已就绪，申请联调】
项目名称：[项目名称]
版本：[版本号]
模块：[模块名]
当前状态：开发中 -> 联调中
接口文档：[路径或链接]
联调环境：[地址]
测试账号：[账号信息]
重点关注：[字段/错误码/性能点]
截止时间：[日期]
```

### 2.2 提测申请模板（前后端发给测试）
```text
【开发完成，申请提测】
项目名称：[项目名称]
版本：[版本号]
模块：[模块名]
当前状态：联调中 -> 提测中
系分文档：[路径或链接]
接口文档：[路径或链接]
测试环境：[地址]
测试账号：[账号信息]
自测结果：[通过项/未通过项]
重点测试场景：[列表]
截止时间：[日期]
```

### 2.3 测试结论模板（测试发给产品+前后端）
```text
【测试结论通知】
项目名称：[项目名称]
版本：[版本号]
模块：[模块名]
当前状态：[提测中 -> 待发布] 或 [提测中 -> 开发中]
测试报告：[路径或链接]
结论：[通过/不通过]
阻塞缺陷：[数量与级别]
上线建议：[建议上线/暂缓上线]
截止时间：[日期]
```

### 2.4 回归修复模板（测试发给责任开发）
```text
【阻塞缺陷回归通知】
项目名称：[项目名称]
版本：[版本号]
模块：[模块名]
当前状态：提测中 -> 开发中
缺陷单：[链接或编号]
阻塞原因：[简述]
期望修复时间：[日期时间]
请修复后重新发起提测。
```

### 2.5 发布确认模板（产品发给全员）
```text
【发布确认通知】
项目名称：[项目名称]
版本：[版本号]
模块：[模块名]
当前状态：待发布 -> 已发布
发布窗口：[日期时间]
发布负责人：[姓名]
回滚方案：[链接]
请各角色确认发布准备完成。
```

## 3. 快速填写示例（宠物寄养场景）
```text
项目名称：边牧小院寄养平台
版本：v1.0.0
模块：在线预约与入住管理
PRD文档：docs/prd/边牧小院_prd_v1.md
原型图：docs/prototype/预约流程_v1.fig
截止时间：2026-04-15
测试环境：https://test.bianmu.com
```

## 4. 规范建议
1. 所有模板消息统一保留项目名称、版本、模块、状态。
2. 每次状态变更必须落文档，避免口头同步。
3. 阻塞缺陷未清零前不得进入发布状态。
