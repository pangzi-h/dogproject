# 多智能体联动协议

## 模板入口
- 快速复制模板请使用：`promot/多agent使用模板.md`

本协议用于统一 产品智能体、前端开发智能体、后端开发智能体、测试智能体 的协作方式，避免信息丢失、状态不同步和责任不清。

## 1. 状态流转
1. `需求中`：产品智能体正在澄清或转换PRD。
2. `方案中`：前后端正在输出系分和技术方案。
3. `开发中`：前后端进入编码和自测。
4. `联调中`：前后端联调并修复问题。
5. `提测中`：测试智能体执行功能与非功能验证。
6. `待发布`：测试通过，形成上线建议。
7. `已发布`：发布完成并归档。

## 2. 角色职责
1. 产品智能体：产出并确认PRD、维护需求边界、统一优先级。
2. 前端开发智能体：前端系分、实现、联调、提测通知。
3. 后端开发智能体：后端系分、实现、接口契约、提测通知。
4. 测试智能体：测分、执行测试、缺陷闭环、测试报告。

## 3. 统一消息头
所有跨Agent消息必须包含以下字段：
- `项目名称`
- `版本`
- `模块`
- `当前状态`
- `目标接收方`
- `文档链接`
- `截止时间`
- `风险说明`

## 5. 缺陷闭环规则
1. 缺陷最小字段：标题、环境、步骤、预期、实际、级别、责任人、状态。
2. 状态流转：`新建 -> 已确认 -> 修复中 -> 待复测 -> 已关闭`。
3. 若存在阻塞缺陷，必须回退到 `开发中` 状态并重新联调或提测。

## 6. 文档归档规则
1. PRD归档：产品智能体维护版本与变更记录。
2. 系分归档：前后端分别维护最新设计文档。
3. 测试归档：测试智能体维护测分和测试报告。
4. 未完成事项：由对应责任智能体更新未完成事项文档。

## 7. 默认门禁（本项目）
1. 优先级体系：P0/P1/P2。
2. 默认技术路线：按各智能体默认配置执行。
3. 发布前置条件：P0缺陷清零，测试结论为通过，文档可追溯。

